# AIDS Center Backend

FastAPI backend for the AIDS Center project.

This repository does not currently define a `Makefile`, `justfile`, or package-manager script aliases. The documented commands below are the direct build and run commands used by the project.

## Stack

- Python 3.14
- FastAPI
- SQLAlchemy async engine
- Alembic migrations
- PostgreSQL 16 via Docker Compose

## Project Entry Point

The main API application lives in `app/main.py` and should be started with:

```bash
uvicorn app.main:app --reload
```

There is also a top-level `app.py`, but it is a minimal hello-world app and is not the primary backend entrypoint for this project.

## Prerequisites

- Python 3.14+
- Docker Desktop or a compatible Docker engine
- `pip`

## Environment Variables

The app reads configuration from `.env`.

Current required variable:

- `DATABASE_URL`

Example format:

```env
DATABASE_URL=postgresql+asyncpg://aidscenter:localdevpassword@localhost:5432/aidscenter
```

The repository already includes a local development `.env`. For production or shared environments, replace development secrets before deployment.

## Install Dependencies

Create and activate a virtual environment, then install Python dependencies:

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## Docker Commands

Docker Compose is used here to run PostgreSQL only. The FastAPI app itself currently runs locally on your machine, not in a container.

Start the database:

```bash
docker compose up -d
```

Stop the database:

```bash
docker compose down
```

Stop the database and remove the Postgres volume:

```bash
docker compose down -v
```

View database logs:

```bash
docker compose logs -f db
```

## Run The API Locally

1. Start PostgreSQL with Docker Compose.
2. Activate your virtual environment.
3. Run Alembic migrations.
4. Start Uvicorn.

Commands:

```bash
source venv/bin/activate
alembic upgrade head
uvicorn app.main:app --reload
```

Default local URLs:

- API: `http://127.0.0.1:8000`
- Swagger UI: `http://127.0.0.1:8000/docs`
- ReDoc: `http://127.0.0.1:8000/redoc`
- Health check: `http://127.0.0.1:8000/health`

## Database Migration Commands

Apply all migrations:

```bash
alembic upgrade head
```

Roll back one migration:

```bash
alembic downgrade -1
```

Create a new autogenerated migration:

```bash
alembic revision --autogenerate -m "describe_change"
```

Create an empty migration:

```bash
alembic revision -m "describe_change"
```

Show current migration version:

```bash
alembic current
```

Show migration history:

```bash
alembic history
```

## API Surface

Current notable routes:

- `GET /health`
- `GET /api/articles`
- `GET /api/articles/featured`
- `GET /api/articles/{slug}`
- `POST /api/articles`
- `PUT /api/articles/{article_id}`
- `DELETE /api/articles/{article_id}`

## Notes

- `docker-compose.yml` provisions PostgreSQL with a named volume called `postgres_data`.
- Alembic loads `DATABASE_URL` from `.env` through `alembic/env.py`.
- SQLAlchemy is configured in async mode, so the database URL must use the `postgresql+asyncpg` driver.
